# ADR-03 – Arquitectura Cliente-Servidor

## Decisión
Utilizar una arquitectura cliente-servidor separando frontend y backend.

## Motivo
Esta arquitectura facilita la escalabilidad del sistema y mejora la organización del código, permitiendo que cada capa evolucione independientemente.

## Consecuencias
Aumenta la complejidad inicial del proyecto, pero facilita el mantenimiento y la incorporación de nuevas funcionalidades a futuro.
